Get 
- Used to retrieve information
- Data is sent via query string
- Information is plainly visible in the URL
- Limited amount of data can be sent 

Post 
- Used to post data to the server
- Used to write/create/update
- Data is sent via request body, not a query string!
- Can send any sort of data (JSON)

REST(Representational state transfer)
- It's an 'arqchitectural style for distributed hypermedia systems'
- It's basically a set of guidelines for how a client + server should communicate
- and perform CRUD operations on a given resource
- The main idea of REST is treating data on the server-side as resourcers that can be CRUDed
- The most common way of approaching REST is in formatting the URLs and HTTP verbs in your applications

CRUD - Create, read, update, delete/destroy
